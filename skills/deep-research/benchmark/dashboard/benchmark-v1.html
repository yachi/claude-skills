Analyst Observations

Key Patterns

The skill's value is structural rigor, not raw intelligence

Both with-skill and baseline reach the same correct conclusions in all 5 evals. The baselines are competent analyses. The skill's value-add is forcing methodology disclosure, adversarial review, assumption audits, confidence calibration, and standards mapping — exactly the structural elements that separate "good enough for internal discussion" from "defensible under expert cross-examination."

Baseline has real strengths the skill should learn from

Across all 5 evals, baselines consistently produced more concise output (43-55% shorter), more developer-friendly guidance (code examples, cookie attribute tables, spare parts lists), and occasionally unique insights the skill missed (Tokio latency warning, Web Worker isolation, Google Consent Mode v2, ISA-95 Purdue model). The skill should incorporate guidance to preserve practical implementation advice alongside rigor.

Hardest eval (Rust vs Zig) showed the narrowest gap

The Rust vs Zig trading eval had the closest scores: 8.5 vs 7.0 cross-examination survivability. The baseline independently produced a weighted decision matrix, named real companies (Citadel, Jane Street, Databento), and identified the Tokio async pitfall. For highly technical topics where the model already has deep domain knowledge, the marginal value of the skill is lower (~0.5 SD technical uplift vs ~1.5 SD regulatory/compliance uplift).

● Skill Consistently Adds

  • Research decomposition / methodology transparency
  • Per-finding confidence percentages (not just "High")
  • Standards cited by clause/section number
  • Adversarial review with structured counterarguments
  • Assumption audit with risk-if-wrong analysis
  • Failure mode enumeration
  • Conditions that change the recommendation
  • Cost/TCO modeling with line items
  • ~85% more source URLs on average

● Baseline Occasionally Beats Skill

  • Code examples and implementation guidance
  • Conciseness (43-55% shorter)
  • Management-friendly communication style
  • Practical operational tips (spare parts, vendor screening checklists)
  • Specific tool recommendations (kani, miri, TrustInSoft)
  • Better signal-to-noise for time-pressed readers

Iteration 2 Recommendations

Skill Improvements

Add implementation guidance to the skill protocol

The baseline consistently produces more actionable implementation advice (code examples, tool names, configuration details). Add a Phase 6.5 "Implementation Guidance" to the skill that says: "After synthesizing findings, include practical next steps — code snippets, tool recommendations, configuration examples — not just what to do but how to do it."

Add conciseness guidance

With-skill outputs average 4,273 words vs baseline's 2,265 words (1.9x). Add guidance: "Use collapsible sections for secondary evidence. Lead with a 3-sentence executive summary. The main body should be under 3,000 words — put supporting detail in appendices."

Raise assertion thresholds

A1 (5+ sources) and A2 (3+ data points) never discriminate. Raise to 15+ sources and 10+ data points. Add new assertions: "Includes cost/TCO analysis" and "States conditions that change the recommendation."

Capture baseline's unique strengths

The baseline produced insights the skill missed in 4/5 evals (Tokio warning, Web Worker isolation, Google Consent Mode v2, ISA-95 Purdue model, spare parts stockpiling). Consider adding: "After completing your research, ask: what practical operational advice would a practitioner add that my standards-focused analysis may have missed?"

Final Verdict

Is the Skill Worth It?

Yes. The /dr skill produces research that survives expert cross-examination (avg 8.7/10 vs 5.8/10 baseline). It costs 1.8x tokens and 1.9x time — a modest premium for transforming "good internal analysis" into "audit-defensible, expert-grade research."

The skill's primary value is structural discipline: methodology transparency, standards compliance mapping, adversarial review, and calibrated confidence. These are exactly the elements that separate research you can act on from research you'll second-guess. The iteration 2 improvements (implementation guidance, conciseness, baseline insights) would close the remaining gaps without sacrificing rigor.
